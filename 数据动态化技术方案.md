# 香港保险对比 Web 应用 — 数据动态化技术方案

> 版本：V8.2 → V9（数据动态化）
> 日期：2026-05-25
> 目标：将数据从静态硬编码改为动态来源，支持定时爬虫更新 + 用户手动刷新

---

## 一、现状分析

### 当前架构
```
index.html（单文件）
  ├── savingsProducts[]     ← 静态 JS 数组（22个产品）
  ├── companies[]          ← 静态 JS 数组（23家公司）
  ├── medicalProducts[]    ← 静态 JS 数组（8个产品）
  ├── vhisProducts[]       ← 静态 JS 数组（13个产品）
  ├── ciProducts[]         ← 静态 JS 数组（13个产品）
  ├── premiumData{}        ← 静态 JS 对象（16个产品年缴数据）
  ├── dividendData[]       ← 静态 JS 数组（23家公司）
  ├── promotions[]         ← 静态 JS 数组（11家公司优惠）
  └── productDetails{}    ← 静态 JS 对象（产品详情弹窗）
```

### 痛点
1. **数据更新成本高**：每次改数据都要编辑 HTML + 重新部署
2. **数据时效性差**：用户反复强调"数据是活的"
3. **无法追溯历史**：没有数据版本管理
4. **团队协作难**：数据散落在单文件里，无法多人维护

## 二、目标架构

### 2.1 整体架构图

```
┌─────────────────┐     ┌──────────────────┐     ┌─────────────────┐
│  定时爬虫服务    │────▶│   JSON 数据文件   │────▶│  Vercel 静态托管 │
│  (Node.js Cron) │     │  /public/data/   │     │  index.html     │
└─────────────────┘     └──────────────────┘     └─────────────────┘
        │                         │                          │
        │ 每天 06:00 HKT       │  Git Commit/Push       │  用户访问
        ▼                         ▼                          ▼
   抓取各保险公司官网          自动触发 Vercel Deploy      fetch(/data/*.json)
   分红实现率 / 产品更新
```

### 2.2 数据分层策略

| 层级 | 数据类型 | 更新频率 | 存储方式 |
|------|---------|---------|---------|
| **静态基准** | 公司档案、产品参数 | 手动 | Git LFS / JSON |
| **半动态** | 储蓄产品 IRR/单利 | 每周 | JSON API |
| **高频动态** | 分红实现率、优惠信息 | 每日 | JSON API + 缓存 |
| **用户生成** | FNA 分析结果 | 实时 | localStorage |

#### 3.3 前端改造（index.html）

**当前**：数据硬编码在 `<script>` 标签内
**改造后**：页面加载时 `fetch()` 获取 JSON

```javascript
// 改造前
const savingsProducts = [ {name:"宏挚传承",...}, ... ];

// 改造后
let savingsProducts = [];
let companies = [];
let metadata = {};

async function loadData() {
  try {
    const [savingsRes, companiesRes, metaRes] = await Promise.all([
      fetch('/data/savings.json?t=' + Date.now()),  // 防止缓存
      fetch('/data/companies.json?t=' + Date.now()),
      fetch('/data/metadata.json?t=' + Date.now())
    ]);
    
    const savingsData = await savingsRes.json();
    const companiesData = await companiesRes.json();
    metadata = await metaRes.json();
    
    savingsProducts = savingsData.products;
    companies = companiesData.companies;
    
    // 更新页面数据版本提示
    updateDataVersionBadge(metadata);
    
    // 渲染表格
    renderSavingsTable(getFilteredSortedSavings());
    renderCompanyCards();
    // ... 其他渲染函数
    
  } catch (err) {
    console.error('数据加载失败，使用内置备份数据', err);
    useFallbackData();  // 内置最小化备份数据
  }
}

// 用户手动刷新按钮
async function userRefreshData() {
  showLoadingSpinner();
  await loadData();
  showToast('数据已更新至 ' + metadata.lastUpdated);
}

// 每5分钟自动检查更新（静默）
setInterval(async () => {
  const res = await fetch('/data/metadata.json?t=' + Date.now());
  const newMeta = await res.json();
  if (newMeta.version !== metadata.version) {
    showUpdateBanner('发现新数据，点击刷新');
  }
}, 5 * 60 * 1000);

document.addEventListener('DOMContentLoaded', loadData);
```

### Phase 2：定时爬虫服务（预计 5-7 天）

#### 2.1 爬虫目标数据源

| 数据类型 | 来源 | 抓取难度 | 更新频率 |
|---------|------|---------|---------|
| 分红实现率 | 各保险公司官网"分红实现率"专栏 | ⭐⭐⭐ | 每年（年报后） |
| 产品 IRR 参数 | 保单说明书 PDF | ⭐⭐⭐⭐⭐ | 产品升级时 |
| 优惠信息 | 官网优惠页面 / 经纪平台 | ⭐⭐ | 每月 |
| 公司评级 | 标普/穆迪官网 API | ⭐ | 评级变动时 |

#### 2.2 爬虫架构（Node.js）

```
crawler/
├── package.json
├── cron.js                # 入口：定时任务调度
├── sources/
│   ├── aia.js            # 友邦官网爬虫
│   ├── manulife.js       # 宏利官网爬虫
│   ├── axa.js            # 安盛官网爬虫
│   └── ...（共23个）
```

#### 2.3 核心爬虫逻辑示例

```javascript
// sources/aia.js
const axios = require('axios');
const cheerio = require('cheerio');
const pdf = require('pdf-parse');

async function crawlAiaDividend() {
  // 1. 访问友邦分红实现率页面
  const res = await axios.get('https://www.aia.com.hk/zh-hk/dividend-fulfillment-ratio');
  const $ = cheerio.load(res.data);
  
  // 2. 解析表格数据
  const dividend2024 = $('table.dividend-table tr').map((i, el) => {
    return {
      product: $(el).find('td').eq(0).text(),
      rate2024: $(el).find('td').eq(1).text()
    };
  }).get();
  
  // 3. 对比上次数据，检测变化
  const oldData = require('../data/dividend.json');
  const changes = detectChanges(oldData.aia, dividend2024);
  
  if (changes.length > 0) {
    console.log('友邦分红实现率有更新：', changes);
    // 4. 更新 JSON 文件
    await fs.promises.writeFile(
      '../data/dividend.json',
      JSON.stringify({...oldData, aia: dividend2024}, null, 2)
    );
    // 5. 提交 Git 并 Push（触发 Vercel 部署）
    await gitCommitAndPush(`更新友邦分红实现率 ${new Date().toISOString()}`);
  }
}

// cron.js - 定时任务
const cron = require('node-cron');

// 每天 06:00 HKT 执行
cron.schedule('0 6 * * *', async () => {
  console.log('开始每日数据爬取...');
  await crawlAiaDividend();
  await crawlManulifeDividend();
  // ... 其他22家
  console.log('每日数据爬取完成');
}, { timezone: 'Asia/Hong_Kong' });
```

#### 2.4 反爬虫策略

各保险公司官网可能有反爬虫措施，需要：
1. **User-Agent 轮换**
2. **请求频率限制**（每家间隔 5-10 秒）
3. **Cookie/Session 管理**
4. **CAPTCHA 处理**（人工辅助或第三方打码平台）
5. **备用方案**：与保险经纪平台（如港险智）合作获取结构化数据

#### 2.5 部署爬虫服务

**选项 A：轻量方案（推荐）**
- 部署在 **Vercel Serverless Function**
- 每天用 GitHub Actions 定时触发
- 成本：免费额度内

**选项 B：稳定方案**
- 部署在 **WorkBuddy 服务器**（用户已有 Windows Server）
- 用 PM2 或 node-cron 管理定时任务
- 成本：无额外费用

```yaml
# .github/workflows/daily-crawl.yml
name: Daily Data Crawl
on:
  schedule:
    - cron: '0 22 * * *'  # UTC 22:00 = HKT 06:00
  workflow_dispatch:        # 支持手动触发

jobs:
  crawl:
    runs-on: ubuntu-latest
    steps:
      - uses: actions/checkout@v3
      - uses: actions/setup-node@v3
        with: { node-version: '24' }
      - run: cd crawler && npm install && node cron.js
      - run: |
          git config user.name "Data Bot"
          git commit -am "自动更新数据 $(date)"
          git push
```

#### 3.3 Service Worker 缓存策略（进阶）

用 Service Worker 实现离线可用 + 后台更新：

```javascript
// sw.js
self.addEventListener('fetch', event => {
  if (event.request.url.includes('/data/')) {
    // 数据请求：网络优先，失败时用缓存
    event.respondWith(
      fetch(event.request).catch(() => caches.match(event.request))
    );
  }
});
```

## 五、风险与应对

| 风险 | 影响 | 应对措施 |
|------|------|---------|
| 保险公司官网改版 | 爬虫失效 | 1) 爬虫失败自动告警；2) 保留上一版本数据 |
| 反爬虫封 IP | 数据更新中断 | 1) 使用代理池；2) 降级到手动更新模式 |
| JSON 数据格式错误 | 前端渲染失败 | 1) JSON Schema 校验；2) 前端 `try-catch` + 备用数据 |
| Vercel 部署失败 | 用户看不到更新 | 1) 部署失败自动回滚；2) 预览 URL 先验证 |
